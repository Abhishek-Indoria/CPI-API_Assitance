CPI / API Calculator – Reference Guide

This guide is derived from the CPI Reference Document provided internally. It explains how CPI and API are applied in contracts and how the CPI / API Calculator should be used to validate pricing.

1. CPI and API – As Defined in the CPI Reference Document

API (Annual Price Increase)

API is a contractually defined fixed percentage increase. As per the CPI Reference Document, API values are agreed at the time of contract signature and remain constant throughout the contract term.

  • API percentage does not change year to year
  • Increase application depends on the contract anniversary
  • Used when price predictability is required

CPI (Consumer Price Index)

CPI is an index-based increase mechanism. As stated in the CPI Reference Document, CPI increases are applied based on the applicable CPI notification for the relevant period.

  • CPI percentage may vary year to year
  • CPI requires validation using official CPI notification letters
  • CPI is applied only on defined anniversaries

2. Increase Types (As Used in the CPI Reference Document)

Flat per Annum

The increase is applied once per contract year, after completion of one full year. No increase is applied in the first contract year.

Flat per Term

The increase is applied only at contract renewal. No annual compounding occurs during the running term.

At Renewal

Increase is applied only when the contract renews and not during the active term.

3. Contract Types Explained

Initial Contract

For an Initial contract, the CPI Reference Document defines:

  • The anniversary is automatically derived from the contract start date
  • No anniversary date input is required from the user
  • The first increase is applied only after completing 12 months

Example:

  • Contract Start Date: 01-07-2020
  • First Anniversary: 01-07-2021
  • No increase before 01-07-2021

Upsell – Co-term

As per the CPI Reference Document, a co-term upsell:

  • Aligns to the parent (initial) contract anniversary
  • Does not create a new anniversary
  • May have a stub period before the next anniversary

Example:

  • Initial Contract Start: 01-07-2020
  • Upsell Start Date: 01-12-2022
  • Parent Anniversary: 01-07
  • Stub Period: 01-12-2022 → 30-06-2023
  • First increase applied on: 01-07-2023

Upsell – Non-co-term

A non-co-term upsell is treated independently, as defined in the CPI Reference Document:

  • The upsell has its own lifecycle
  • The anniversary is derived from the upsell start date
  • Increase applies only after completing 12 months from upsell start

Example:

  • Upsell Start Date: 01-02-2023
  • First Anniversary: 01-02-2024

4. How to Use the CPI / API Calculator

  1. Select Contract Nature (Initial or Upsell)
  2. If Upsell, select Co-term or Non-co-term
  3. Enter the Initial Contract Start Date
  4. Enter the Upsell Start Date (only for upsell)
  5. Select CPI or API and the applicable increase type
  6. Enter Base Annual Price and Increase Percentage
  7. Validate the calculated value against ISP

5. CPI Notification Letters

CPI values must be validated using the official CPI notification letters:

  • CPI Notification – PC
  • CPI Notification – VB

This guide must be used together with the CPI Reference Document. In case of conflict, the CPI Reference Document takes precedence.
